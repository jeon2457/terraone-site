이 코드를 만든이유는 깃허브(https://jeon2457.github.io/terraone-site/)에서 만든 웹페이지가 경로(C:\GitHub\terraone-site\public)를 제대로 찾아들어가지 못해서 페이지가 열리지를 않아서 속에 들어있는 모든 데이타를 그대로두고 복사해서 바깥 루트폴더((C:\GitHub\terraone-site)로 붙여넣기해서 이중으로 데이타를 가지고있다.  여기까지는 작동하는데 아무런 문제가 없지만 만약에,
데이타 수정시 양쪽모두에 데이타를 수정(복사_붙여넣기)를 해야하는 번거러움이 생긴다.
그래서 간편하게 이 스크립트코드를 만들어서 자동처리하기위해 만든것이다. 
public/에서 웹사이트 수정을하면 자동으로 처리가된다. ===>  C:\GitHub\terraone-site\public 에서만 수정작업!!

"sync-pages" 스크립트가 public 폴더 → 루트 폴더 동기화 역할




이제 이후부터는

public/에서 웹사이트 수정 →

npm run sync-pages →

Git add/commit/push → GitHub Pages 업데이트

firebase deploy --only hosting → Firebase Hosting 업데이트

흐름 그대로 진행하면 됩니다.

★★★   Firebase Hosting 정적 사이트 코드 배포   ★★★

사이트주소:  https://terraone-d0318.firebaseapp.com/

Firebase Hosting은 정적 사이트 배포가 목적이라서, 소스 코드를 직접 브라우저에서 수정하는 방식이 아니고 로컬에서 편집 후 배포하는 방식입니다.

cd C:\GitHub\terraone-site
npm run sync-pages          # public → 루트 (GitHub Pages용)
git add .
git commit -m "Update site"
git push origin main        # GitHub Pages 반영
firebase deploy --only hosting  # Firebase Hosting 반영
